    How a browser turns HTML + CSS into pixels

    The task list above exists because the browser ran this pipeline once for this document, and re-runs parts of it every time script.js touches the DOM.

    1. 01

      Parsing

      The browser reads the raw HTML byte stream and turns it into a stream of characters, then structural nodes.

    2. →
    3. 02

      Tokenization

      The character stream is broken into tokens — start tags, end tags, attribute names, text — which the parser consumes one at a time.

    4. →
    5. 03

      DOM tree

      Tokens are assembled into the Document Object Model: a tree of nodes that JavaScript can read and mutate.

    6. →
    7. 04

      CSSOM tree

      CSS goes through the same parse–tokenize cycle, producing the CSS Object Model — every rule, with specificity resolved.

    8. →
    9. 05

      Render tree

      DOM and CSSOM are combined into the render tree: only visible nodes, each carrying its final computed style.

    10. →
    11. 06

      Layout & paint

      The browser computes the geometry of every render tree node, then paints pixels to the screen.
